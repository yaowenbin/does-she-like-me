AI判断对方喜欢程度 Skill 设计方案

核心定位：AI通过捕捉「对方行为信号、语言信号、情绪信号、互动信号」，结合恋爱可得性底层逻辑（心理、行为、社交等），量化对方喜欢程度（0%-100%），同时给出可解释的判断依据，避免“凭感觉”，实现数据化、精准化判断。

设计核心原则：信号可捕捉、维度可量化、逻辑可追溯、结果可落地，拒绝模糊化判断，每个Skill都对应“具体信号+判断规则+权重占比”，适配AI的数据分析和模式识别能力。

一、核心底层 Skill（基础必备，占比40%）

核心作用：搭建AI判断的基础框架，基于“喜欢的本质=主动投入+情绪倾斜+长期稳定”，覆盖最核心、最易捕捉的信号，是判断的核心依据。

Skill 1：主动互动频次与质量（权重15%）

核心逻辑：喜欢的核心表现是“主动”，但需区分“无效主动”（敷衍）和“有效主动”（有内容、有投入），结合行为心理学的“强化机制”——越喜欢，主动频率越高、质量越高。

具体判断规则（AI可捕捉信号）：

- 主动发起聊天/见面的频次：每周主动≥3次（高分）、1-2次（中等）、0次（低分）；

- 主动发起的内容质量：是否有具体话题（如“今天看到一家店很适合你”）、是否关注你的日常（如“你今天加班累吗”），而非敷衍式“在吗”“吃了吗”；

- 主动延续话题的能力：你回复简短时，对方是否能主动拓展话题，而非“哦”“哈哈”终结对话；

- 特殊节点主动：生日、节日、你情绪低落时，是否主动关心、准备小细节（非群发祝福）。

AI落地：通过用户上传的聊天记录、互动记录，提取“主动发起方”“话题内容”“回复时长”“延续次数”等数据，自动匹配评分。

Skill 2：情绪投入与共情能力（权重15%）

核心逻辑：基于普通心理学的“情绪价值供给”，喜欢一个人会本能关注其情绪，愿意投入情绪成本，共情对方的喜怒哀乐，而非“事不关己”。

具体判断规则（AI可捕捉信号）：

- 情绪回应及时性：你分享开心/难过时，是否在1小时内回应（紧急情绪），且回应不敷衍；

- 共情深度：是否能get到你的情绪点（如你说“加班好累”，对方是“加油”还是“辛苦啦，要不要我给你点杯奶茶”）；

- 情绪同步：对方是否会主动分享自己的情绪（如“今天上班好烦，想跟你说说”），把你当作情绪出口；

- 负面情绪包容：你闹小脾气、情绪低落时，对方是耐心安抚，还是不耐烦、冷处理。

AI落地：通过自然语言处理（NLP）分析聊天记录中的情绪词、语气词、回应内容，判断情绪投入程度；结合用户标注的“自己的情绪状态”，匹配对方的回应质量。

Skill 3：边界感与排他性（权重10%）

核心逻辑：基于进化心理学的“排他性需求”，喜欢一个人会主动划清与其他异性的边界，减少不确定性，让对方感受到“被重视、被专属”。

具体判断规则（AI可捕捉信号）：

- 异性社交边界：是否主动告知你“和异性朋友的相处”（如“今天跟同事聚餐，有男有女”），而非隐瞒；

- 拒绝暧昧：是否明确拒绝其他异性的示好，且不搞“养鱼”行为（如不回复异性的暧昧消息、不单独和异性去私密场所）；

- 公开意愿：是否愿意把你介绍给朋友、家人，或在社交平台暗示你的存在（非强制公开，重点是“主动意愿”）；

- 专属感：是否有只对你做的事（如只给你发晚安、只陪你聊天到深夜）。

AI落地：通过聊天记录中“异性相关话题”的提及频率、回应态度，结合用户输入的“对方异性社交情况”，判断排他性得分。

二、进阶判断 Skill（提升精准度，占比35%）

核心作用：弥补“底层Skill”的不足，捕捉“隐性喜欢信号”（非直白表达，却能体现心意），结合社会心理学、沟通语言学的逻辑，减少误判。

Skill 4：行为一致性与长期稳定性（权重10%）

核心逻辑：短期的主动可能是“新鲜感”，长期的、稳定的行为才是“真喜欢”，避免AI被“短期热情”误导（如追求期的刻意讨好，后续冷淡）。

具体判断规则（AI可捕捉信号）：

- 行为连贯性：3个月内，主动频次、情绪投入、回应质量是否稳定（波动不超过30%），而非“忽冷忽热”；

- 承诺兑现率：对方说过的小事（如“明天陪你聊天”“给你带好吃的”），兑现率≥80%（高分）、50%-80%（中等）、＜50%（低分）；

- 主动规划未来：是否主动提及“和你的未来”（如“周末我们去看电影吧”“下次带你去吃那家你说过的店”），而非只谈当下。

AI落地：通过用户长期上传的互动数据，建立“行为趋势模型”，分析3个月内的信号波动，判断稳定性得分。

Skill 5：语言细节与潜意识信号（权重10%）

核心逻辑：基于沟通语言学、符号互动论，人在表达时会不自觉流露潜意识，语言细节（用词、语气、称呼）能反映真实心意，比“直白表白”更可信。

具体判断规则（AI可捕捉信号）：

- 专属称呼：是否有只对你的称呼（如小名、专属昵称），而非通用的“兄弟”“朋友”；

- 用词倾向：是否频繁使用“我们”“一起”“以后”等带有“绑定感”的词，而非“你”“我”分开表述；

- 语气细节：聊天时是否用温柔、亲昵的语气词（如“呀”“啦”“呢”），而非全程生硬、冷漠；

- 主动询问细节：是否主动问你的喜好、习惯（如“你喜欢吃辣吗”“你平时休息喜欢做什么”），并记住（如下次聊天提及“你说过你不喜欢吃香菜”）。

AI落地：通过NLP分析聊天记录中的用词、语气、称呼，提取“绑定感词汇”“专属词汇”“细节询问频次”，自动评分。

Skill 6：投入成本与优先级（权重10%）

核心逻辑：基于博弈论+经济学的“成本-收益”逻辑，喜欢的程度和“愿意投入的成本”正相关，对方会把你放在“优先级”位置，而非“可有可无”。

具体判断规则（AI可捕捉信号）：

- 时间成本：对方是否愿意牺牲自己的时间陪你（如加班后仍陪你聊天、放弃自己的娱乐活动陪你做你喜欢的事）；

- 精力成本：是否愿意为你花心思（如记住你的喜好、为你解决问题、在你需要时随叫随到）；

- 优先级：当你和其他事情冲突时（如对方朋友约聚会，你需要陪伴），对方是否优先选择你；

- 物质成本（非必需）：是否愿意为你花小钱（如买一杯奶茶、一个小礼物），重点是“心意”，而非金额。

AI落地：通过用户输入的“对方投入情况”（时间、精力、物质），结合聊天记录中的“优先级表述”，量化投入得分。

Skill 7：社交背书与态度公开（权重5%）

核心逻辑：基于社会心理学的“群体背书”，喜欢一个人会愿意让自己的社交圈知道你的存在，通过“公开态度”强化对你的重视。

具体判断规则（AI可捕捉信号）：

- 朋友层面：是否主动带你见自己的好朋友，或在朋友面前提及你（如“我朋友说想认识你”）；

- 社交平台：是否在朋友圈、抖音等平台，隐晦或直接提及你（如发和你相关的照片、配文“今天很开心”）；

- 家人层面：是否愿意和你聊自己的家人，或有“带你见家人”的规划（长期判断）。

AI落地：通过用户输入的“对方社交公开情况”，结合聊天记录中“朋友/家人相关话题”的提及，判断背书得分。

三、辅助校正 Skill（降低误判率，占比25%）

核心作用：规避“单一信号误判”（如对方本身话少，不是不喜欢；对方主动，只是性格外向），结合人格心理学、生态系统视角，对判断结果进行校正，提升精准度。

Skill 8：人格与性格校正（权重10%）

核心逻辑：不同人格的人，表达喜欢的方式不同，需结合大五人格特质，校正判断标准，避免“以偏概全”。

具体校正规则（AI可调整权重）：

- 内向型人格：主动频次可能低，但主动质量高、稳定性强，需提高“主动质量”“稳定性”的权重，降低“主动频次”的权重；

- 外向型人格：主动频次高，但可能对多人主动，需重点判断“排他性”“专属感”，避免误判为“喜欢”；

- 神经质高的人：情绪波动大，可能偶尔冷处理，需结合“长期稳定性”，避免因短期情绪波动误判；

- 宜人性高的人：对谁都温和，需区分“普遍温和”和“对你的特殊温和”（如只对你花心思）。

AI落地：让用户先输入“对方的大致人格特质”（可选，如内向/外向、情绪稳定/敏感），AI自动调整各Skill的权重，进行校正。

Skill 9：场景与关系阶段校正（权重10%）

核心逻辑：不同关系阶段（初识、暧昧、追求中、稳定相处），喜欢的表达形式不同；不同场景（异地/同城、线上/线下），信号捕捉难度不同，需针对性校正。

具体校正规则：

- 初识阶段：重点看“主动频次”“语言礼貌度”，不苛求“情绪投入”“排他性”；

- 暧昧/追求阶段：重点看“情绪投入”“专属感”“投入成本”，稳定性可适当降低权重；

- 稳定相处阶段：重点看“稳定性”“承诺兑现率”“未来规划”，主动频次可适当降低权重；

- 异地场景：重点看“线上互动质量”“情绪回应及时性”“主动规划见面”，弱化“线下陪伴”的权重；

- 同城场景：重点看“线下见面主动频次”“见面时的肢体语言”（用户可上传描述）。

AI落地：让用户选择“当前关系阶段”“相处场景”，AI自动调整各Skill的评分标准和权重。

Skill 10：反向排除 Skill（权重5%）

核心逻辑：排除“假性喜欢”信号（如对方只是寂寞、需要情绪价值、想利用你），避免AI误判，结合恋爱可得性中的“风险预判”。

具体排除规则（出现以下情况，扣除对应分数，严重时直接判定为“低喜欢”）：

- 只在自己需要时找你（如深夜emo、需要帮忙时），你需要时却冷处理；

- 聊天只聊自己，从不关注你的需求，拒绝分享自己的真实情况；

- 频繁暧昧，但从不明确关系，回避“喜欢”“在一起”等话题；

- 有明显的“利用倾向”（如向你借钱、让你帮忙做事，做完后冷淡）。

AI落地：通过NLP分析聊天记录中的“需求导向”“话题倾向”，结合用户输入的“异常情况”，自动扣除分数，校正判断结果。

四、AI Skill 落地执行流程（关键）

1. 用户输入基础信息：对方人格特质（可选）、当前关系阶段、相处场景；

2. 用户上传数据：聊天记录（文本/截图识别）、互动记录（主动频次、见面情况等）、异常情况描述；

3. AI执行各Skill：按“底层→进阶→辅助校正”的顺序，提取信号、量化评分、调整权重；

4. 计算总得分：加权求和（底层40%+进阶35%+辅助25%），得出0%-100%的喜欢程度；

5. 输出结果：给出具体得分、等级（如0-20%：无喜欢；21-40%：有好感；41-60%：喜欢；61-80%：很喜欢；81-100%：深爱），同时列出“核心判断依据”（如“主动频次高，但排他性不足，扣除XX分”）；

6. 动态更新：用户持续上传新的互动数据，AI重新执行各Skill，更新得分和判断依据。

五、Skill 优化建议（长期迭代）

- 增加“肢体语言识别”（如用户上传见面照片/视频，AI识别眼神接触、肢体开放度、距离感），补充线下信号；

- 引入“用户反馈校正”：用户对AI判断结果进行“准确/不准确”评价，AI结合反馈调整各Skill的权重和判断规则；

- 细化“性别差异”：结合性别研究，调整各Skill的判断标准（如男生表达喜欢更偏向“行动”，女生更偏向“情绪分享”）；

- 增加“时间维度”：按“7天、30天、90天”分别计算得分，展示喜欢程度的变化趋势，帮助用户判断“是升温还是降温”。

补充说明：所有Skill的“权重”可根据用户需求调整（如用户更看重“排他性”，可提高Skill3的权重）；AI捕捉的信号均为“可量化、可验证”，避免主观臆断，确保判断结果的可信度和落地性。

六、工程架构决策（关键补充）

结论：不建议二选一，建议采用“独立Skill核心包 + Agent编排集成”的双层架构。

- 独立Skill核心包负责“可复用、可测试、可解释”的评分引擎（你的底层驱动逻辑）；
- Agent层负责“对话交互、数据采集、调用核心包、结果讲解与追问”；
- 这样既保留了底层能力的稳定性，又能让上层产品快速迭代。

为什么不是“全塞进Agent代码”：

- 判断逻辑会和Prompt、会话状态、工具调用耦合，后续难以做A/B测试和回归验证；
- 规则变更后难以定位影响范围，不利于版本管理（尤其是权重和阈值变更）；
- 解释性输出容易不稳定，难保证“同输入同输出”。

为什么不是“只做独立Skill包”：

- 缺少用户引导与上下文追问，数据不完整时准确率会下降；
- 无法利用Agent做动态澄清（如关系阶段、人格特征缺失时自动补问）；
- 无法形成完整产品体验（趋势追踪、建议话术、复盘报告）。

推荐分层（可直接落地）：

1) domain-core（独立包，底层驱动）  
- 输入：结构化信号（主动频次、回应时延、情绪共情分等）；  
- 能力：10个Skill评分、权重校正、反向排除扣分、总分聚合；  
- 输出：score(0-100)、level、reasons、risk_flags、trend_delta；  
- 特点：纯函数化、无UI无对话依赖、可单元测试。

2) signal-extractor（可独立模块）  
- 聊天文本/截图OCR转结构化信号；  
- NLP提取“专属称呼、绑定词、共情回应、承诺兑现”等特征；  
- 产出标准化SignalSchema，供domain-core消费。

3) agent-orchestrator（你的Agent主工程）  
- 负责问答流程：补齐缺失变量（人格、阶段、场景）；  
- 调用signal-extractor + domain-core；  
- 生成“分数 + 证据 + 下一步建议”的对话式输出；  
- 负责记忆与动态更新（7/30/90天趋势）。

4) policy-safety（策略层）  
- 输出中避免绝对化结论，使用概率区间与不确定性声明；  
- 增加反操控提醒（借钱、情绪索取、长期暧昧不确认关系）；  
- 对高风险关系给出优先级提示而非“继续投入”建议。

推荐目录草案：

- /packages/domain-core  
- /packages/signal-extractor  
- /apps/agent-runtime

- /configs/thresholds

数据契约建议（先定协议，再写实现）：

- InputSignal  
  - profile: 人格特征（可空）  
  - relation_stage: 初识/暧昧/追求中/稳定相处  
  - context_scene: 同城/异地/线上主导/线下主导  
  - metrics: 各Skill可观测指标（频次、时延、兑现率等）  
  - anomalies: 反向排除事件列表

- ScoringResult  
  - total_score: number  
  - level: string  
  - skill_scores: Record<skill_id, score>  
  - weight_snapshot: Record<skill_id, weight>  
  - deductions: 扣分明细  
  - reasons: Top证据列表  
  - confidence: 置信度（由数据完整性决定）

建议迭代节奏（从雏形到可用）：

- V0（最快可用）：先做规则引擎 + 手动录入结构化数据；  
- V1（产品可用）：接入聊天文本解析，支持自动特征提取；  
- V2（增强准确）：接入用户反馈闭环，做权重微调与阈值学习；  
- V3（差异化）：引入多模态（语音语气/线下行为描述）与趋势预测。

一句话决策：

把“喜欢程度判断”做成独立Skill核心包，把“用户交互与流程编排”放在Agent里，这才是你这个产品长期可维护、可扩展、可验证的工程形态。
