# 「她爱你嘛.skills」开源发布方案书

> **目的**：定义一个可上架 GitHub 的 **Agent Skill**（**她爱你嘛.skills**），在自愿提供的聊天记录上，输出**多透镜、可复核**的解读报告——从**演化与基因隐喻**、**心理学**、**叙事与欲望的话语**到**星座文化叙事**，形成「从生物到心理到文学到星座」的立体分析；**本质仍是可读信号 + 替代解释的加权**，不是算命或基因判决。  
> **工程对标**：本地仓库中的 [ex-skill](https://github.com/therealXiaomanChu/ex-skill)（`SKILL.md` frontmatter、`prompts/` 分文件、可选 `tools/`、`reference.md` 可维护词典），便于社区理解安装方式与演进节奏。  
> **文档版本**：2026-04-02（v2 · 多透镜扩展）。  
> **实现位置**：`skills/does-she-like-me/`（`SKILL.md`、`prompts/`、`tools/`），仓库 `README.md` / `INSTALL.md`。

---

## 一、产品一句话

把聊天记录当作**互动样本**，用**多层透镜**（演化生物学隐喻、亲缘与性选择、心理学依恋与互惠、文学性的「欲望的翻译」、星座作为**自愿文化标签**）分别生成**平行段落**，再在 **`synthesis.md` 规则**下收敛为：**综合区间判断 + 原文证据 + 不确定性与反例 + 可验证的下一步**；语气可对齐「bro」，内核保持尊重、标注每种说法的**认识论强度**。

## 二、与优秀开源项目的对齐方式（可照抄的工程习惯）

| 借鉴自 ex-skill 等成熟 Skill | 在本项目中的落点 |
| --------------------------- | ---------------- |
| `SKILL.md`：frontmatter、`user-invocable`、触发词列表 | 首屏写清触发与输出结构；声明多透镜与**非科学预测** |
| `prompts/` 分文件、主流程引用路径 | **按透镜拆分** + `intake` + `synthesis` + `safety` |
| `tools/` 可选、解析与隐私 | 脱敏、分条、未来可接微信导出文本（与 ex-skill 解析思想一致，代码不强行复制） |
| `reference.md` 长期维护 | **信号词典、误判库、演化/心理学术语极简释义**（避免模型乱用术语） |
| 安全边界硬规则区块 | 骚扰/跟踪/未成年人/隐私：**与 ex-skill「安全边界」同级醒目** |

**不要求** fork ex-skill；**要求** README **致敬**其 Skill 工程结构与双层记忆思路，并说明本项目场景为「好感解读」而非「人格蒸馏」。

## 三、多透镜分析框架（核心产品差异）

以下为**分析报告中的固定模块**；每一透镜必须遵守 **§四 认识论层级**，并在段落首行标注 **【透镜标签 + 强度】**。

### 3.1 透镜 A：演化生物学 · 达尔文式叙事（隐喻层）

**用途**：用**本科普级、非个体差异判决**的语言，讨论聊天行为在「信号—成本—互惠」框架下**可能**如何被理解。

| 子话题 | 分析要点（绑定聊天记录证据） |
| ------ | ---------------------------- |
| **性选择 / 信号成本**（达尔文传统、Zahavi 等 costly signaling 的通俗版） | 「主动发起、持续投入、情绪劳动」若稳定出现，可视为**高成本信号的一种可能解读**；若仅为低成本附和，则**不可**直接推出「深爱」。 |
| **互惠与博弈** | 回复节奏、是否双向提问、资源（时间/注意力）分配，用**互惠假说**讨论「像不像长期合作博弈里的均衡」，须列**替代解释**（忙、敷衍、职业化礼貌）。 |
| **親投資/親代撫養**（概念层面） | **禁止**套用到具体个人做道德评判；仅可用于讨论「承诺词汇 vs 行为一致性」时的**抽象提醒**。 |

**硬性约束**：不得声称「演化决定了她爱不爱你」；必须使用 **「作为一种叙事/类比」** 表述。

### 3.2 透镜 B：基因与《自私的基因》（隐喻层，非基因检测）

**用途**：借用 Dawkins **复制子 / 载体**、**延伸表现型**等概念，讨论「人在聊天里传递的是策略还是真心」——**永远是修辞与思想实验**，不是生物学检测。

| 子话题 | 分析要点 |
| ------ | -------- |
| **复制子 vs 载体** | 聊天中的「话术模板、emoji 习惯」可能是社会模因的复制；有趣处在于：**偏离模板的真实脆弱披露**往往信息量更高（用证据句支撑）。 |
| **自利与合作的相容** | 短期的「自私基因」叙事可对照「长期互利、依恋系统」的心理现实；输出必须写清 **两种叙事可能同时成立**。 |
| **禁止** | 不说「她的基因更偏向如何如何」；不做任何**个体**层面的基因决定论。 |

建议在 `reference.md` 中给 **2～4 句**《自私的基因》语境说明，防止模型望文生义。

### 3.3 透镜 C：心理学（证据强度相对更高，仍非诊断）

**用途**：用**教科书级通俗**概念绑定聊天证据；**不提供诊断**，不贴病理标签给第三方。

建议覆盖（可按 `prompts/lens_psychology.md` 勾选）：

| 模块 | 与聊天的绑定 |
| ---- | ------------- |
| **依恋理论**（安全/焦虑/回避/混乱，仅作沟通风格谱系） | 回避式简短回复 vs 焦虑式追问，须对照**具体句子**；说明同一行为可被不同依恋风格解释。 |
| **爱情三角/亲密—激情—承诺**（Sternberg 等，科普版） | 文本中能识别的是**亲密与承诺的言语线索**；激情**不可**从纯文字强推。 |
| **互惠、社会交换、自我决定理论（自主/胜任/关系）** | 是否支持对方的自主、是否贬低、是否单向索取关注。 |
| **认知偏差提醒** | 确认偏误、读心术、锚定最近一条消息等——专门一节写给用户本人，不诛心对方。 |

### 3.4 透镜 D：文学与「欲望的翻译」（话语与叙事层）

**用途**：把聊天当作**微型文本**，分析**欲望与好感如何被编码进修辞**（不引用低俗语录堆砌）。

| 子话题 | 分析要点 |
| ------ | -------- |
| **欲望的翻译** | 「她想说什么」与「字面说什么」的缝隙：省略、反话、试探、撤回、表情包替代表达；每条需**悬置**多种译本：关心/敷衍/害羞/边界。 |
| **叙事位置** | 她把自己放在叙事里的哪种角色（照顾者、玩伴、被追求者、平等朋友）；是否**允许双人故事延续**。 |
| **互文与梗** | 内部梗、引用作品，往往拉高**共同叙事资本** —— 仍可能是友谊。 |

可提示阅读趣向（非必读书单）：恋人絮语类话语分析、现代小说中「未说出口的欲望」母题——**仅作文案品味**，不作学历要求。

### 3.5 透镜 E：星座（文化叙事层 · 强制弱强度）

**用途**：仅当用户提供**血型/星座/MBTI 等自愿标签**时启用；作为 **ex-skill 式标签系统** 的延伸，与 [ex-skill README 中的星座/MBTI 支持](https://github.com/therealXiaomanChu/ex-skill) 精神一致：**影响「沟通风格的翻译规则」而非命运预测**。

| 规则 | 说明 |
| ---- | ---- |
| **科学声明** | 开篇固定一句：**占星并非科学**，以下为**文化修辞与共通刻板印象的反思性使用**。 |
| **用法** | 把星座当作「她**可能**熟悉的自我叙事语言」：例如双子式跳跃 vs 天蝎式试探——**全部**落回**具体聊天句**验证；不匹配则说明「标签失效」。 |
| **禁止** | 太阳星座决定爱不爱；水逆背锅；任一星座「一定渣/一定专一」。 |

若用户未提供星座，本透镜输出 **「未提供标签，跳过；若补充可重跑」**，不主动索要真实生日侵犯隐私。

## 四、认识论层级与输出标注（必须写进 SKILL.md）

所有报告在 **§六 综合交付物** 之前，先给一张 **「透镜强度说明」** 小表：

| 层级 | 标签 | 含义 |
| ---- | ---- | ---- |
| L1 | `心理学-科普` | 较贴近经验科学与临床外推，仍有误判 |
| L2 | `演化-隐喻` | 思想实验与类比，**非**个体预测 |
| L3 | `基因-修辞` | 讨论模因/策略，**非**基因检测 |
| L4 | `文学-阐释` | 文本细读，多重译本并存 |
| L5 | `星座-文化` | 自愿标签下的修辞，**非**科学 |

每一段分析句尾建议可加 **（证据：引用原话）** 或 **（推断；待证实）**，与 ex-skill 强调原材料证据的精神一致。

## 五、伦理、边界与免责声明（发布必备 · 与 v1 兼容并加强）

在 v1 基础上**增补**：

1. **多透镜不得叠 Buff**：禁止把星座 + 演化 + 某句聊天「三重确认」成绝对结论；**synthesis** 必须做**冲突调解**（各透镜结论矛盾时怎么看）。  
2. **反 PUA / 反操控**：不提供利用心理弱点操控对方的「必胜话术」。  
3. 其余条款（非心理咨询、非监视、反对性别刻板印象、安全拒答等）保持不变。

## 九、核心能力设计（更新）

### 9.1 输入（Intake）

在 v1 基础上增加 **可选**：

- 对方与用户（或仅对方）**自愿提供的**星座/MBTI/标签；**可不提供**。  
- 用户希望我们**优先侧重**哪一透镜（默认「全开启」）。

### 9.2 行为层量表（`score_rubric.md`，与透镜正交）

保留主动性、情绪信号、一致性、冲突处理、替代解释等 **Likert + 引文**；该层为「聊天行为学」底座，**合成时不得被星座覆盖**。

### 9.3 最终交付物（模板 v2）

1. **透镜强度说明表**（§四）。  
2. **各透镜小节**（每节带 L1–L5 标签 + 证据/推断标注）。  
3. **行为层维度表**（v1 表格）。  
4. **Synthesis**：  
   - 若多透镜冲突，**优先采信绑定直接证据更强的陈述**；  
   - 给出 **「喜欢程度」区间**（如：偏低/不明/中等偏高）+ **置信度文字**；  
   - **禁止**输出单一百分比「她 87% 爱你」类数字（易误导）。  
5. **「最像的三种叙事」**（含纯友谊版本）。  
6. **可验证的下一步** + **何时停止过度解读**。

## 十、Prompt 策略要点（更新）

- **分块**与**证据卡片**：先抽结构化「时间-发言人-引文」，再喂各透镜，减少胡编。  
- **星座模块独立开关**：未提供标签时 `lens_astrology.md` 输出固定短句。  
- **基因/演化关键词消毒**：在 `lens_gene_rhetoric.md` 顶部列 **禁用表述清单**（如「她的基因选择了你」）。  
- **文学透镜防油腻**：禁止大段摘抄情色网文腔；聚焦**修辞与省略**。  
- **语气**：bro 仅限外层；透镜正文保持可读、略学术俏皮亦可，**不物化**。

**结语**：你要的「从生物到心理学到文学到星座」的丰富度，应通过 **分透镜并列 + 认识论分级 + 合成时的自我纠错** 实现；这样既对得起 **达尔文与道金斯的思想趣味**，也对得起 **心理学与用户的现实关系**，更不让 **星座从娱乐升格为判决**——这才是可长期放在 GitHub 上的好东西。
