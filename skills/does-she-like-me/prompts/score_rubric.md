# score_rubric · 行为层量表（与透镜正交）

## 说明

本层描述**可观察互动**，用 1–5 分 + **每条必须绑定** `证据卡片` 中的引文。1=弱/少/未见，5=强/多/稳定出现。`替代解释`必须用一句友好人话：给出“除了喜欢/不喜欢以外，还可能是什么原因”。  
若证据不足，写 `nc`（not enough coverage）并说明缺什么。

## 维度表

输出：

```markdown
## 行为层量表（L1-行为观察）

| 维度 | 分(1-5或nc) | 引文证据 | 替代解释 |
| ---- | ----------- | -------- | -------- |
| 主动性（发起/延续话题） | | 「…」 | |
| 互惠性（双向提问、接梗） | | | |
| 情绪劳动（安抚、共情、祝福） | | | |
| 亲密线索（分享日常/脆弱/昵称） | | | |
| 一致性与尊重（贬损、操控、画饼） | | | 低分也可能是开玩笑语境 |
| 冲突与修复（冷处理 vs 建设性） | | | |
```

## 注意

* 「尊重」维度：**嘲讽、贬低、物化、威胁** 若出现，须在 synthesis 中升为风险信号。
* 行为层分数**不可**被星座透镜覆盖或推翻；合成时以**直接引文**优先。

## Core-10 信号矩阵（强制输出）

目标：把“喜欢程度”拆成可复核子能力，避免单维误判。

计分规则：

- 分值：`1~5` 或 `nc`  
- 每项必须给 `证据` + `替代解释`  
- 无证据不得给高分  
- 如涉及人格/阶段/场景校正，必须同步更新 `weight_snapshot`

默认权重：

- Skill1~3（底层）：0.40  
- Skill4~7（进阶）：0.35  
- Skill8~10（校正）：0.25

### 输出模板（Markdown）

```markdown

| Skill | 名称 | 分(1-5或nc) | 权重 | 引文证据 | 替代解释 |
| ----- | ---- | ----------- | ---- | -------- | -------- |
| S1 | 主动互动频次与质量 | | 0.15 | 「…」 | |
| S2 | 情绪投入与共情能力 | | 0.15 | | |
| S3 | 边界感与排他性 | | 0.10 | | |
| S4 | 行为一致性与长期稳定性 | | 0.10 | | |
| S5 | 语言细节与潜意识信号 | | 0.10 | | |
| S6 | 投入成本与优先级 | | 0.10 | | |
| S7 | 社交背书与态度公开 | | 0.05 | | |
| S8 | 人格与性格校正 | | 0.10 | | |
| S9 | 场景与关系阶段校正 | | 0.10 | | |
| S10 | 反向排除（风险扣分项） | | 0.05 | | |
```

### 计算约定

- `S10` 为扣分项：若命中利用/操控/只在索取时出现等模式，允许输出低分并在 synthesis 单独解释。  
- 如用户未提供人格或关系阶段：`S8/S9` 可写 `nc`，同时降低置信度。  
- 仅在跨时间样本充分时，`S4` 可评为高分。

### Structured Summary 对齐

在结尾结构化小节中写：

- `skill_scores`: `{S1: x, ..., S10: x}`  
- `weight_snapshot`: `{S1: w, ..., S10: w}`  
- `deductions`: 来自 `S10` 的扣分理由数组
